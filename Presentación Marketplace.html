Contexto del Proyecto

El Problema

  • Sin catálogo estructurado de perfiles profesionales locales
  • Contratos informales sin trazabilidad del estado del trabajo
  • Baja visibilidad frente a Upwork y Fiverr (plataformas extranjeras)
  • Sin mecanismos de calificación ni reputación
  • Dependencia de plataformas que ignoran el contexto económico regional
Propuesta de valor:

Una plataforma propia, contextualizada al ecosistema CORHUILA, que permita el registro de freelancers, publicación de servicios, contratación trazable y calificación post-proyecto.

Historias de Usuario

HU-01 · Registrar Perfil de Freelancer

ALTA prioridad 8 Story Points Sprint 1
Como administrador de la plataforma, quiero registrar un nuevo freelancer con su información profesional y habilidades técnicas, para que pueda publicar servicios en el marketplace y ser contratado.
Criterios de Aceptación (5)
CA-01 · Validación campo obligatorio — Dado el formulario → cuando guarda sin 'nombre' → entonces muestra "El nombre es obligatorio" y no guarda.
CA-02 · Documento duplicado — Dado ya existe el doc. → cuando registra otro igual → entonces error "El documento ya está registrado".
CA-03 · Registro exitoso — Dado campos válidos → cuando hace clic en Guardar → entonces aparece en el listado inmediatamente.
CA-04/05 · Habilidades — Agregar "React" → aparece en lista y puede eliminarse individualmente. Al eliminar, solo esa desaparece.
Historias de Usuario

HU-04 · Gestionar Solicitudes Recibidas

ALTA prioridad 8 Story Points Sprint 3
Como freelancer, quiero ver, aceptar o rechazar las solicitudes de contratación, para que solo trabajar con clientes y proyectos que me interesen.
Criterios clave
CA-02 · Solo el freelancer dueño del servicio puede aceptar o rechazar.
CA-03 (RF09) · Al aceptar: el sistema crea automáticamente un proyecto en estado PENDIENTE. Operación atómica.
¿Qué pasa si falla la creación del proyecto?

La transacción se revierte. La solicitud vuelve a estado PENDIENTE. No queda ningún proyecto incompleto en la base de datos.

HU-05 · Seguimiento del Proyecto

ALTA prioridad 13 SP — Mayor complejidad Sprint 3
Como freelancer, quiero actualizar el estado del proyecto y que el sistema registre automáticamente las fechas, para que clientes y administradores puedan rastrear el avance.
Máquina de estados
PENDIENTE
Sistema (auto) → EN_PROGRESO
Freelancer · fechaInicio ✓ → EN_REVISIÓN
Freelancer → COMPLETADO
Freelancer · fechaFin ✓
Desde cualquier estado: CANCELADO · fechaFin ✓ · sin calificación
Solo proyectos COMPLETADOS habilitan la calificación (RF12, Sprint 4). Los estados COMPLETADO y CANCELADO son irreversibles.
Modelado BPMN

P1 · Contratación de Servicio

🛒 CLIENTE
🟢 INICIO → Busca y filtra servicios (RF06) → Selecciona servicio → Redacta mensaje → Envía solicitud (RF07) → Solicitud creada en estado PENDIENTE
💼 FREELANCER
Recibe notificación de solicitud → Revisa solicitud → ◆ ¿Acepta?
⚙ SISTEMA
[SÍ] → Crea proyecto PENDIENTE automáticamente (RF09) + vincula a solicitud → [NO] → Registra rechazo, no crea proyecto → 🔴 FIN
Gateway ◆ en BPMN: El rombo representa la bifurcación del flujo según una condición Sí / No. Los eventos de inicio son 🟢 y las tareas del sistema son ⚙.
Modelado BPMN

P2 · Gestión del Estado del Proyecto

⚙ SISTEMA
🟢 INICIO → Proyecto creado automáticamente en PENDIENTE (RF09) → Disponible en el panel del freelancer
💼 FREELANCER
Cambia a EN_PROGRESO → (Sistema registra fechaInicio) → Realiza trabajo → Cambia a EN_REVISIÓN → ◆ ¿Completar o cancelar? → COMPLETADO / CANCELADO
⚙ SISTEMA
Registra fechaFin automáticamente (RF11)
🛒 CLIENTE
◆ ¿Proyecto COMPLETADO? → [SÍ] Ingresa puntuación 1-5 + comentario (RF12) → [NO/CANCELADO] Sin opción de calificar → 🔴 FIN
Modelado BPMN

P3 · Registro de Freelancer y Publicación

👤 ADMINISTRADOR
🟢 INICIO → Ingresa datos del freelancer (RF01) → ◆ ¿Datos válidos y únicos? → [SÍ] Guarda / [NO] Corrige → Agrega habilidades: React, Figma… (RF02) → ◆ ¿Nueva categoría? → [SÍ] Crea (RF03) / [NO] Usa existente
⚙ SISTEMA
Valida UNIQUE en documento y correo (BD) → Guarda freelancer + habilidades en PostgreSQL → Freelancer disponible para publicar servicios
💼 FREELANCER
Crea servicio: título, descripción, precio, tiempo (RF05) → Selecciona categoría → ◆ ¿Precio y tiempo positivos? → [SÍ] Publica / [NO] Corrige → Activa servicio (isActive = true) → 🔴 FIN — Servicio visible para clientes
